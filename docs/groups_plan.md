# Multiple Fuchsian Groups — Plan

## Goal

Let the user switch between four Fuchsian group presets. Both the tessellation (Fund. Domain mode) and Dirichlet domain computation use the currently selected group. A new group selector appears in the UI alongside the existing mode selector.

---

## Groups

| Label | Definition | Generator(s) | Element type |
|-------|-----------|--------------|--------------|
| PSL(2,ℤ) | Modular group | S: z↦−1/z,  T: z↦z+1 | elliptic (order 2, 3), parabolic, hyperbolic |
| ⟨z↦2z⟩ | Cyclic hyperbolic dilation | g: z ↦ 2z | purely hyperbolic |
| ⟨z↦z+1⟩ | Cyclic parabolic translation | T: z ↦ z+1 | purely parabolic |
| ⟨S, T²⟩ | Theta group | S: z↦−1/z,  T²: z↦z+2 | elliptic (order 2) + parabolic |

### Fundamental domains (canonical shape)

| Group | FD shape |
|-------|----------|
| PSL(2,ℤ) | Geodesic triangle {|Re z|≤½, |z|≥1} — 3 boundary arcs |
| ⟨z↦2z⟩ | Half-annulus {1/√2 ≤ |z| ≤ √2} — 2 boundary arcs |
| ⟨z↦z+1⟩ | Vertical strip {|Re z|≤½} — 2 boundary arcs |
| ⟨S, T²⟩ | Larger geodesic polygon (index-3 in PSL(2,ℤ)) — 4+ boundary arcs |

### Updated `_mat_key`

Extend to handle float matrices (rounding to 8 decimal places) while preserving behaviour for integer matrices:

```python
def _mat_key(M):
    flat = np.round(M.flatten().astype(float), 8)
    if flat[0] < -1e-9 or (abs(flat[0]) < 1e-9 and flat[1] < -1e-9):
        flat = -flat
    return tuple(flat)
```

### Fixed `dirichlet_boundary`

Current code skips only the identity matrix. Problem: some non-identity elements fix z₀ (e.g., S fixes i in PSL(2,ℤ)), causing a degenerate bisector. Fix: skip any image that coincides with z₀:

```python
# was:
if _mat_key(M) == _mat_key(I): continue
# becomes:
w = mobius(M, z0)
if abs(w - z0) < 1e-6 or w.imag <= 1e-9: continue
```

## Tessellation strategy per group

**PSL(2,ℤ)** (`use_std_fd=True`): keep current approach — transform the 3 hardcoded arc endpoints via each γ and re-sample using `sample_geodesic_H`. Gives the classic triangular tiling.

**All other groups**: Dirichlet-based approach:
1. Compute `fd_arcs = dirichlet_boundary(canonical_z0, elements)` once — returns list of sampled arc arrays.
2. For each γ in elements, apply γ pointwise: `transformed = mobius(M, arc_array)`.
3. Keep only points where `Im > 1e-9`, draw in both panels.

This is uniform and works for any Fuchsian group.

## Element-type feature

Every non-identity element of PSL(2,ℝ) belongs to exactly one of three classes, determined by its trace:

| Type | Condition | Fixed points | Geometric action |
|------|-----------|--------------|------------------|
| **Elliptic** | \|tr M\| < 2 | one point *inside* ℌ | rotation around that point |
| **Parabolic** | \|tr M\| = 2 | one point on ∂ℌ = ℝ∪{∞} | translation along horocycles |
| **Hyperbolic** | \|tr M\| > 2 | two points on ∂ℌ | translation along the axis geodesic |

### New math

```python
def element_type(M):
    """Return 'elliptic', 'parabolic', or 'hyperbolic' for a 2×2 PSL(2,R) matrix."""
    t = abs(np.trace(M))
    if t < 2 - 1e-9:   return 'elliptic'
    if t < 2 + 1e-9:   return 'parabolic'
    return 'hyperbolic'

def fixed_points(M):
    """Return fixed point(s) of M as complex numbers.
    Elliptic: one point in ℌ (and its conjugate below).
    Parabolic: one real point (or ∞ if c=0).
    Hyperbolic: two real points."""
    a, b, c, d = M[0,0], M[0,1], M[1,0], M[1,1]
    if abs(c) < 1e-12:                    # upper-triangular: fixes ∞
        return [np.inf]
    # solve cz² + (d-a)z - b = 0
    disc = (a - d)**2 + 4*b*c             # = tr(M)² - 4·det(M)
    z1 = ((a - d) + np.sqrt(complex(disc))) / (2 * c)
    z2 = ((a - d) - np.sqrt(complex(disc))) / (2 * c)
    return [z1, z2]
```

### Visual feature: tile coloring by element type

In Fund. Domain mode, color each tile by the type of the group element that generates it:

| Type | Color |
|------|-------|
| Identity (tile 0 = FD itself) | neutral grey |
| Elliptic | red family |
| Parabolic | blue family |
| Hyperbolic | green family |

Each family cycles through shades as depth increases, so tiles of the same type at different depths remain distinguishable.

### Visual feature: fixed-point markers

A "Show fixed points" toggle button (shown in Fund. Domain mode). When active:
- **Elliptic fixed points** — star marker (★) inside ℌ
- **Parabolic fixed points** — triangle marker on the real axis (or at ∞: arrow)
- **Hyperbolic fixed points** — circle markers on the real axis

Markers appear in both panels (converted via Cayley for the disk).

### Description panel: element types section

In Fund. Domain and Dirichlet modes, the description panel gains a fourth line:

```
Element types in <group label>:
  elliptic: <k_e> elements  (|tr| < 2,  rotation about a point in ℌ)
  parabolic: <k_p> elements  (|tr| = 2,  horocyclic translation, fixes ∞ or a real point)
```

Counts are for the currently enumerated set at the current depth.
